Weekly Planning

Summary

This describes Dendron's weekly planning process. Think of it as sprint retrospectives, Dendron style.

Process

Before

  • @weeklyPlanner (Private)
    • create weekly.journal.{date}
    • create weekly.journal.2021.11.16.plan (this will create a note with the planning checklist lead)
  • @everyone (Private): copy the weekly planning user user template and go through checklist

During

  • during the meeting, we recommend that those following along have two windows of the google doc open
    • one window to follow along the current section
    • another window to add any Next items or Discussion you want to bring up based on whats currently being discussed

After

  • @weeklyPlanner (Private): go through After Meeting Checklist
  • @everyone (Private): weekly Journal

Update Next Week Milestones

  1. Create a new milestone
  • version is one minor version above current version (eg. if we're currently on v55, next week would be v56)
  • due date is Tuesday of next week
  1. Move tasks from Up Next Milestone into next weeks milestones
  2. Assign owners to these tasks

Update This Week Triage

  1. For all issues from oncall that are on triage, add it to the Triage section of weekly planning

Update CROP

  • See Crop Community Issue (Private)
  1. Announce in the #feedback channel the CROP of this week. Tag the original submitter in the response if possible
  2. Update Crop Tasks (Private)
  • add voted on CROP to active
  • move other crops to the bottom of Previous (Private)
  1. Make sure there 3 CROP items for Next (Private)
