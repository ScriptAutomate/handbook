Onboarding

Summary

The following describes the onboarding process at Dendron

Process

Prerequisites

  1. Send Kevin your github information
  2. Setup Dendron
    • NOTE: we recommend using VSCode Insider as your Dendron Client and regular VSCode for development work
  3. Get permission to join the github workspace (you should get a notification on Github when this is ready)
  4. Go throught the Dendron Tutorial. When you install Dendron for the first time, you will be taken through a tutorial that walks you over the basics. If you missed it, you can bring it up again by bringing up the Command Prompt (Private) and typing Dendron: Launch Tutorial
    • IMPORTANT: do this before proceeding.

Steps

Setting Up Your Workspace

  1. Clone the Dendron private workspace 

    git clone git@github.com:dendronhq/org-workspace.git
    • NOTE: If you are onboarding via our trial, please use the following command 
    git clone ssh://github.com/dendronhq/org-workspace-incubator

  2. Open up your Dendron workspace

    • NOTE: we're using code-insider CLI in this example but this will vary depending on what VSCode client you use
    cd {workspace}
code-insider .

  3. You will see a prompt to Open workspace file on the bottom right corner. Click on the prompt

  4. Wait ~30s for your workspace to initialize. Dendron is initializing your workspace and downloading company documents at this point

  5. After your workspace is finished initializing, you should have a workspace with the following vaults (your workspace might look different depending if you are full time or still in the trial period)

Start individual onboarding

  1. Your manager should already have setup a hierarchy for you. You can find it by going to user.{YOUR_ALIAS}
  2. Go to user.{YOUR_ALIAS}.onboarding where you should have a copy of our Onboarding checklist to walk you through the rest of the process
